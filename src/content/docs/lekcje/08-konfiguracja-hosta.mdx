## Czym jest DHCP

**DHCP** (*Dynamic Host Configuration Protocol*) — protokół przydzielania adresów IP i parametrów
konfiguracji sieci klientom automatycznie. Klient łączy się z siecią, *dzierżawi* od serwera adres
na zadeklarowany czas (*lease time*), po którym odnawia umowę lub zwalnia adres.

Komputer-klient otrzymuje od serwera DHCP:

- **adres IP** (z puli zarządzanej przez serwer)
- **maskę podsieci**
- **bramę domyślną** (gateway)
- **adresy serwerów DNS**
- **opcjonalne dodatki**: nazwa domeny, NTP, route'y statyczne, MTU

Bez DHCP musielibyśmy konfigurować każdą maszynę ręcznie. W sieciach domowych DHCP zwykle
biegnie na routerze; w sieciach firmowych — na dedykowanym serwerze (Windows Server, ISC DHCP,
dnsmasq, Kea).

## ISC DHCP Server na Ubuntu

Pakiet `isc-dhcp-server` to klasyczna implementacja referencyjna od Internet Systems Consortium.

```bash
sudo apt install isc-dhcp-server
```

:::caution[ISC DHCP jest deprecated od 2022]
ISC ogłosił **end-of-life dla ISC DHCP w 2022**. Następca:
[**Kea DHCP**](https://www.isc.org/kea/) — modularny, API-driven, REST + JSON config.
W 2026 nowe wdrożenia powinny używać Kea zamiast `isc-dhcp-server`.
:::

### Konfiguracja `/etc/dhcp/dhcpd.conf`

```text title="/etc/dhcp/dhcpd.conf"
default-lease-time 600;
max-lease-time 7200;

option domain-name "lab.local";
option domain-name-servers 8.8.8.8, 1.1.1.1;

subnet 10.10.10.0 netmask 255.255.255.0 {
    range 10.10.10.100 10.10.10.200;
    option routers 10.10.10.1;
    option broadcast-address 10.10.10.255;
}

# Rezerwacja po MAC
host laptop-szefa {
    hardware ethernet 08:00:27:12:34:56;
    fixed-address 10.10.10.50;
}
```

### Interfejs nasłuchu — `/etc/default/isc-dhcp-server`

```text
INTERFACESv4="enp0s8"
```

### Uruchomienie

```bash
sudo systemctl enable --now isc-dhcp-server
sudo systemctl status isc-dhcp-server
journalctl -u isc-dhcp-server -f      # logi na żywo
```

Plik `/var/lib/dhcp/dhcpd.leases` zawiera aktualne dzierżawy.
